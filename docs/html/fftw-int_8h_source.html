247 /*
248  *
249  Things you should define to include your own clock:
250 
251  fftw_time -- the data type used to store a time
252 
253  extern fftw_time fftw_get_time(void);
254  -- a function returning the current time. (We have
255  implemented this as a macro in most cases.)
256 
257  extern fftw_time fftw_time_diff(fftw_time t1, fftw_time t2);
258  -- returns the time difference (t1 - t2).
259  If t1 < t2, it may simply return zero (although this
260  is not required). (We have implemented this as a macro
261  in most cases.)
262 
263  extern double fftw_time_to_sec(fftw_time t);
264  -- returns the time t expressed in seconds, as a double.
265  (Implemented as a macro in most cases.)
266 
267  FFTW_TIME_MIN -- a double-precision macro holding the minimum
268  time interval (in seconds) for accurate time measurements.
269  This should probably be at least 100 times the precision of
270  your clock (we use even longer intervals, to be conservative).
271  This will determine how long the planner takes to measure
272  the speeds of different possible plans.
273 
274  Bracket all of your definitions with an appropriate #ifdef so that
275  they will be enabled on your machine. If you do add your own
276  high-precision timer code, let us know (at fftw@fftw.org).
277 
278  Only declarations should go in this file. Any function definitions
279  that you need should go into timer.c.
280  */
