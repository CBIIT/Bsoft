38/* The header information is stored in four short integers (16 bits each).
39Each integer is stored with the high byte first (Motorola or 'big-endian' format).
40The image data is stored row by row as a continuous stream of data (i.e. no line feeds,
41carriage returns, or end of file control characters are present in the file).
42The amount of storage space that each pixel in the image occupies depends on the
43data type. The header and the image data are stored in the binary format
44(not in the ASCII format).
45
46The format for the small header image file is described below.
47
48Field Position Length
49Width of image 0 2 bytes (integer)
50Height of image 2 2 bytes (integer)
51Bytes per pixel 4 2 bytes (integer)
52Encoded data type6 2 bytes (integer)
53Image data 8 Width * height * bytes per pixel
54
55The encoding of the data type (at position 6) is shown below.
56
57Value Data Type
581 2 byte signed integer
592 Floating point IEEE standard 754
603 8 byte complex floating point (real, imaginary)
615 Packed complex
626 1 byte unsigned integer
637 4 byte signed integer
648 4 byte RGB
659 1 byte signed integer
6610 2 byte unsigned integer
6711 4 byte unsigned integer
6812 8 byte double
6913 16 byte complex double (real, imaginary)
7014 1 byte binary data
71*/
